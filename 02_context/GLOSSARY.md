# GLOSSARY

- Alert:
  Outbound notification generated by the radar. Strong alerts go to Telegram.

- B-shot:
  Project method with two major agentic goals: planning/specs first, implementation second.

- Deduplication:
  Logic that prevents the same listing from being treated as new on every run.

- Digest:
  Email summary of medium-interest candidates, price drops, and run results.

- Dry-run:
  Full pipeline execution without sending real outbound alerts.

- Estimated total cost:
  Price plus shipping plus approximate import/risk adjustments used for decision-making.

- Listing:
  Marketplace item observed from a source such as eBay.

- Mock mode:
  Execution using local fixtures instead of live external services.

- Noop mode:
  Execution where outbound adapters write to console/file instead of Telegram or email.

- Radar:
  Internal system that watches configured marketplaces and alerts only on likely opportunities.

- Risk flag:
  Explicit reason to distrust, penalize, or reject a listing.

- Score:
  Deterministic 0-100 opportunity rating with positive reasons, negative reasons, risk flags, and recommendation.

- Source client:
  Adapter that fetches raw listings from a marketplace API.

- Source normalizer:
  Adapter that converts raw source payloads into the internal listing model.
